# Feature: Utah Startup Map

The following plan should be complete, but validate documentation and codebase patterns before implementing. Read every file listed under CONTEXT REFERENCES before writing a single line of code.

Pay special attention to: the `useDeepgram` hook interface, the data cleaning requirements from the CSV, and the Mapbox style configuration approach for the futuristic aesthetic. All UI uses inline styles — no Tailwind in page-level components.

## Feature Description

An interactive map of ~255 Utah startups at `/map`. Default state is a futuristic 2D top-down view of Utah — near-transparent black land, green (#2a5e49) state border outline, dim road network — so the map blends seamlessly into the page's black background. Clicking a startup marker triggers a cinematic 3D flydown with real Mapbox buildings, an animated info panel, and a slow orbit. A floating Deepgram-powered voice filter lets users say "show me Series A FinTech companies" to filter markers. Non-matching markers dim to 12% opacity rather than disappearing.

The data schema and routing structure are designed to support Phase 2: self-service company profiles, claim/verify flow, job postings, and photo galleries — without requiring schema changes.

## User Story

As an investor, founder, or ecosystem builder,
I want to explore a beautiful interactive map of Utah's startup ecosystem,
So that I can discover companies, understand the landscape geographically, and filter by stage, size, and sector with my voice.

## Problem Statement

Utah has 255+ startups building innovative products, but there's no visual, explorable way to understand the ecosystem — who's building what, where, and at what stage. A map makes the ecosystem viscerally impressive and discovery-driven.

## Solution Statement

A Mapbox GL JS map with a custom dark/transparent style (black land fill matches page background = "see-through" effect, #2a5e49 Utah border outline), custom HTML startup markers with company logos, cinematic 3D flydown on click, and a Deepgram + Claude voice filter system — all state managed by Zustand.

### Task 7: CREATE `scripts/geocode-startups.ts`

This script reads `data/mapdata.csv`, geocodes each address via Mapbox Geocoding API v6, and writes `public/startups.json`. It handles all CSV data quality issues.

```ts
// Run with: npx tsx --env-file=.env.local scripts/geocode-startups.ts
import { parse } from "csv-parse/sync";
import { readFileSync, writeFileSync, existsSync } from "fs";
import path from "path";

const TOKEN = process.env.NEXT_PUBLIC_MAPBOX_TOKEN!;
if (!TOKEN) throw new Error("NEXT_PUBLIC_MAPBOX_TOKEN is required");

// Load existing output to allow incremental re-runs (don't re-geocode what's already done)
const outputPath = path.resolve("public/startups.json");
const existing: Record<string, { lat: number; lng: number }> = {};
if (existsSync(outputPath)) {
  const prev = JSON.parse(readFileSync(outputPath, "utf-8")) as Array<{ linkedin_url: string; lat: number; lng: number }>;
  prev.forEach((s) => { existing[s.linkedin_url] = { lat: s.lat, lng: s.lng }; });
  console.log(`Loaded ${prev.length} existing geocodes`);
}

const csv = readFileSync("data/mapdata.csv", "utf-8");
const rows = parse(csv, { columns: true, skip_empty_lines: true, relax_quotes: true });

function trim(v: unknown): string {
  return String(v ?? "").trim();
}

function normalizeLinkedIn(url: string): string {
  // Strip query params and trailing slashes; normalize to https://
  try {
    const u = new URL(url.startsWith("http") ? url : `https://${url}`);
    return `https://www.linkedin.com${u.pathname.replace(/\/$/, "")}`;
  } catch {
    return url.trim();
  }
}

function getSlug(linkedin_url: string): string {
  const match = linkedin_url.match(/\/company\/([^/?#]+)/);
  return match?.[1] ?? "";
}

function normalizeDomain(website: string): string {
  const w = website.trim();
  if (!w) return "";
  try {
    const url = new URL(w.startsWith("http") ? w : `https://${w}`);
    return url.hostname.replace(/^www\./, "");
  } catch {
    return w.replace(/^(https?:\/\/)?(www\.)?/, "").split("/")[0];
  }
}

function normalizeStage(s: string): string {
  const v = s.trim();
  const map: Record<string, string> = {
    "pre-seed": "Pre-Seed", "seed": "Seed",
    "series a": "Series A", "series b": "Series B+",
    "series b+": "Series B+", "series b ": "Series B+",
    "series c": "Series B+", "series d": "Series D+",
    "series d+": "Series D+",
  };
  return map[v.toLowerCase()] ?? v;
}

function normalizeEmployees(s: string): string {
  const v = s.trim();
  // "201-500" and "200+" both mean large — normalize to "200+"... actually keep as-is
  // since FilterCriteria in store uses the raw values for filtering
  return v;
}

async function geocode(address: string): Promise<{ lat: number; lng: number } | null> {
  const q = encodeURIComponent(address);
  const url = `https://api.mapbox.com/search/geocode/v6/forward?q=${q}&country=us&limit=1&access_token=${TOKEN}`;
  const res = await fetch(url);
  if (!res.ok) { console.error("Geocode error:", res.status, address); return null; }
  const data = await res.json() as { features: Array<{ geometry: { coordinates: [number, number] } }> };
  const coords = data.features?.[0]?.geometry?.coordinates;
  if (!coords) { console.warn("No result for:", address); return null; }
  return { lng: coords[0], lat: coords[1] };
}

const sleep = (ms: number) => new Promise((r) => setTimeout(r, ms));

const results = [];
let geocoded = 0, skipped = 0, failed = 0;

for (const row of rows) {
  const linkedin_url = normalizeLinkedIn(trim(row["LinkedIn Link (map it to Links to get the logo)"]));
  const name = trim(row["Startup Name"]);
  const address = trim(row["Full Address"]);
  const description = trim(row["Description of startup"]).replace(/\n+/g, " ");
  const website = trim(row["Website"]);
  const stage = normalizeStage(trim(row["Stage"]));
  const employees = normalizeEmployees(trim(row["# of Employees"]));
  const section = trim(row["Section"]);
  const domain = normalizeDomain(website);
  const slug = getSlug(linkedin_url);

  let lat: number, lng: number;
  if (existing[linkedin_url]) {
    ({ lat, lng } = existing[linkedin_url]);
    skipped++;
  } else if (address) {
    const geo = await geocode(address);
    if (geo) {
      ({ lat, lng } = geo);
      geocoded++;
    } else {
      lat = 39.321; lng = -111.093; // Utah centroid fallback
      failed++;
    }
    await sleep(120); // ~8 req/sec — within Mapbox's rate limit
  } else {
    lat = 39.321; lng = -111.093;
    failed++;
  }

  results.push({
    slug,
    linkedin_url,
    name,
    address,
    lat,
    lng,
    description,
    website,
    domain,
    logo_url: domain ? `https://logo.clearbit.com/${domain}` : "",
    stage,
    employees,
    section,
  });
}

writeFileSync(outputPath, JSON.stringify(results, null, 2));
console.log(`Done. Geocoded: ${geocoded}, Skipped (existing): ${skipped}, Failed (centroid): ${failed}`);
console.log(`Output: ${outputPath} (${results.length} entries)`);
```

### Task 8: CREATE `app/api/map/parse-filter/route.ts`

```ts
import { NextResponse } from "next/server";
import Anthropic from "@anthropic-ai/sdk";

const SYSTEM_PROMPT = `You extract filter criteria from a user's spoken request about Utah startups.
Return ONLY a JSON object: { "stage": string[], "size": string[], "section": string[] }.
Valid stage: "Pre-Seed", "Seed", "Series A", "Series B+", "Series D+".
Valid size: "1", "2-10", "11-50", "51-200", "200+", "201-500".
Valid section: "B2B Software", "FinTech", "Security", "Bio/Medical Tech", "Energy", "Consumer", "Marketplaces".
Empty array = no filter on that dimension. Return nothing except the JSON object.`;

export async function POST(req: Request) {
  const { transcript } = await req.json() as { transcript: string };
  if (!transcript?.trim()) {
    return NextResponse.json({ stage: [], size: [], section: [] });
  }

  const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  const msg = await anthropic.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 256,
    system: SYSTEM_PROMPT,
    messages: [{ role: "user", content: transcript.slice(0, 500) }],
  });

  const text = msg.content[0]?.type === "text" ? msg.content[0].text.trim() : "{}";
  try {
    const parsed = JSON.parse(text) as { stage?: string[]; size?: string[]; section?: string[] };
    return NextResponse.json({
      stage: parsed.stage ?? [],
      size: parsed.size ?? [],
      section: parsed.section ?? [],
    });
  } catch {
    return NextResponse.json({ stage: [], size: [], section: [] });
  }
}
```

- **GOTCHA**: `transcript` is user voice input — truncate to 500 chars before sending to Claude to prevent prompt stuffing (parallel to `ai-context/SECURITY.md` guidance).
- **VALIDATE**: `curl -X POST http://localhost:3000/api/map/parse-filter -H 'Content-Type: application/json' -d '{"transcript":"show me seed stage fintech companies"}' | jq .` — should return `{ "stage": ["Seed"], "size": [], "section": ["FinTech"] }`

## FUTURISTIC MAP AESTHETIC — DETAILED NOTES

The "see-through / transparent" effect is achieved without canvas transparency:

1. **Land color = page background** → `colorSurface: "#010101"` in Standard config (land fill = near-black). Result: land area is visually indistinguishable from the black page → map appears transparent.

2. **Utah green border** → custom GeoJSON layer with `line-color: #2a5e49, line-width: 2.5`. The bright green border "outlines" Utah against the black void, creating the glowing border aesthetic.

3. **Ghost road network** → Standard config `colorRoads: "#0a0a0a"` — roads are barely visible as dark lines inside Utah. At zoom <10 they're invisible; zoom in and they appear subtly.

4. **3D buildings on flydown** → `setConfigProperty("basemap", "show3dObjects", true)` — buildings materialize in the darkened scene. Combined with fog (`FOG_3D`), depth is dramatic.

5. **Glass morphism UI** → Info panel uses `backgroundColor: rgba(0,0,0,0.92)` + `backdropFilter: blur(16px)` → panels feel like frosted dark glass, enhancing the futuristic aesthetic without colored backgrounds.

6. **Night lighting** → `lightPreset: "night"` — no sunlight on 3D buildings; accent lighting only. This is the most dramatic Mapbox preset and fits the futuristic visual perfectly.

## NOTES

**Why `react-map-gl` v7 and not v8?** v7 is battle-tested with mapbox-gl v3. v8 is newer and may have rough edges. Use `react-map-gl@7` explicitly in the install command.

**Why not Zustand with Context API?** Zustand is simpler and avoids Provider wrapping — ideal for a feature-level store. The existing project has no Zustand, so this introduces it cleanly at the map feature boundary.

**Why inline styles (not Tailwind) for map UI?** All existing page-level components use inline styles. The map UI chrome (info panel, filter chips, voice button) follows the same pattern for visual consistency.

**On `setConfigProperty` for Standard style:** The Mapbox Standard style config properties listed in `mapConfig.ts` come from Mapbox docs. Some may not have exact key names matching — the implementation wraps each call in try/catch so unsupported keys fail silently. At runtime, verify in the browser console that the dark aesthetic loads. If certain properties don't apply (e.g., `colorSurface` doesn't exist in Standard's API), add a fallback using `map.setPaintProperty('land', 'fill-color', '#000000')` after load.

**Phase 2 extensibility:** The `Startup` type includes optional fields for hiring, job postings, photos, and claimed_at. The `slug` field enables future routing to `/map/company/[slug]`. When Phase 2 begins, the data source moves from static JSON to a Supabase `startups` table with the same column names — no API route changes needed.

**Confidence Score:** 8/10 — the main uncertainty is whether Mapbox Standard's `setConfigProperty` keys exactly match what's documented (they've changed between versions). The fallback `setPaintProperty` approach handles this gracefully.
